# Bilibili Music Downloader - 产品需求文档

## 1. 项目概述

一个包含 Web 前端界面的音乐下载工具。用户在网页上输入歌曲名称后，自动在哔哩哔哩（Bilibili）平台搜索该歌曲，从搜索结果中选取匹配的视频，下载视频并通过 ffmpeg 提取音频并转换为 MP3 格式。同时保留命令行方式作为备选入口。

## 2. 目标用户

- 需要从 Bilibili 下载歌曲音频的用户
- 批量下载音乐用于离线收听的用户

## 3. 功能需求

### 3.1 Web 前端界面

提供浏览器访问的 Web 页面，用户通过网页完成全部操作，无需使用命令行。

#### 3.1.1 搜索输入区

- 页面顶部包含一个搜索输入框和搜索按钮
- 输入框支持输入歌曲名称作为搜索关键词
- 支持回车键触发搜索（与点击搜索按钮等效）
- 搜索过程中输入框和按钮显示 loading 状态，防止重复提交

#### 3.1.2 搜索结果展示区

- 搜索结果以列表/卡片形式展示在输入框下方
- 每条结果包含以下信息：
  - 视频封面缩略图
  - 视频标题
  - UP 主名称
  - 视频时长
  - 播放量
  - "下载"操作按钮
- 无搜索结果时显示空状态提示（如"未找到相关视频，请尝试其他关键词"）
- 搜索失败时显示错误提示信息

#### 3.1.3 下载进度区

- 用户点击某条结果的"下载"按钮后，该条目进入下载状态
- 显示实时下载进度条（百分比 + 已下载/总大小）
- 下载完成后显示"转换中..."状态
- 转换完成后显示"下载 MP3"按钮，用户可点击直接下载 MP3 文件到本地
- 整个过程支持同时下载多个视频（队列形式）

#### 3.1.4 页面布局与样式

- 页面采用单页应用（SPA）结构，所有操作在同一页面完成
- 响应式布局，适配桌面端（1024px+）和平板端（768px+）
- 整体风格简洁现代，以哔哩哔哩品牌色（#00A1D6）为主题色
- 页面标题和 logo 显示在顶部

### 3.2 歌曲搜索

- 前端将搜索关键词发送至后端 API
- 后端调用 Bilibili 搜索 API 获取视频列表
- 返回结构化的搜索结果数据供前端渲染

### 3.3 视频下载

- 根据用户选择的视频，获取视频流地址（包含视频流和音频流）
- 下载视频流和音频流到本地临时目录
- 支持下载进度显示

### 3.3 音频提取与转换

- 调用本地 ffmpeg，将下载的视频流和音频流合并（Bilibili 高清视频的音视频是分离的）
- 从合并后的视频中提取音频轨道
- 将音频转换为 MP3 格式（码率 320kbps）
- 清理临时文件（原始视频流、音频流、合并后的视频文件）

### 3.4 后端 API

后端提供 RESTful API 供前端调用：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/search?keyword=xxx` | GET | 搜索视频，返回结果列表 |
| `/api/download/<bvid>` | POST | 触发下载任务，返回任务 ID |
| `/api/progress/<task_id>` | GET | 查询下载/转换进度 |
| `/api/file/<task_id>` | GET | 下载生成的 MP3 文件 |

### 3.5 输出管理

- 支持自定义输出目录，默认为当前目录下的 `output/` 文件夹
- MP3 文件名格式：`{歌曲名} - {UP主}.mp3`
- 如果输出目录不存在，自动创建

### 3.5 额外功能

- 支持通过命令行参数直接指定搜索关键词，跳过交互式输入
- 支持批量搜索下载（可选，后续版本）
- 下载失败时给出明确的错误提示

## 4. 非功能需求

### 4.1 依赖要求

**后端：**
- Python 3.8+
- ffmpeg（需预先安装在系统中并加入 PATH）
- 以下 Python 库：
  - `requests` — HTTP 请求
  - `yt-dlp` — Bilibili 视频下载（支持音视频分离流）
  - `flask` — 提供 REST API 和静态文件服务
  - `flask-cors` — 跨域支持（开发阶段前后端分离时使用）

**前端：**
- Vue 3（通过 CDN 引入，无需构建工具）
- 或纯 HTML + CSS + JavaScript（轻量方案，无需额外框架）

### 4.2 运行环境

- Windows 10/11
- 需要网络连接

### 4.3 性能要求

- 搜索结果在 3 秒内返回
- 下载速度取决于用户网络带宽
- 音频转换应在 10 秒内完成（对于常规时长的歌曲）

## 7. 交互流程

### 7.1 Web 端流程（主要入口）

```
用户在搜索框输入歌曲名称，点击搜索或按回车
       │
       ▼
  前端调用 /api/search 接口
       │
       ▼
  展示搜索结果列表（封面 / 标题 / UP主 / 时长 / 播放量 / 下载按钮）
       │
       ▼
  用户点击某条结果的 [下载] 按钮
       │
       ▼
  前端调用 /api/download 接口，获取任务 ID
       │
       ▼
  轮询 /api/progress 接口，显示下载/转换进度
       │
       ▼
  完成后显示 [下载MP3] 按钮
       │
       ▼
  用户点击下载，浏览器保存 MP3 文件
```

### 7.2 命令行流程（备选入口）

```
用户输入歌曲名称

  搜索 Bilibili
       │
       ▼
  展示搜索结果列表（序号 / 标题 / UP主 / 时长 / 播放量）

       ▼

       │

       │
       ▼
  输出完成，显示文件路径
```

## 8. 技术方案概要

### 8.1 搜索模块

- 调用 Bilibili 搜索接口 `https://api.bilibili.com/x/web-interface/search/type?search_type=video&keyword={keyword}`
- 解析返回的 JSON 数据，提取视频列表信息

### 8.2 下载模块

- 使用 `yt-dlp` 库处理 Bilibili 视频下载，自动处理登录态、CDN 选址、音视频流合并
- `yt-dlp` 原生支持 Bilibili，能正确处理分 P 视频和音视频分离流

### 8.3 音频转换模块

- 调用系统 `ffmpeg` 命令行工具
- 转换命令示例：`ffmpeg -i input.mp4 -vn -ab 320k -ar 44100 -y output.mp3`

### 8.4 后端服务

- 使用 Flask 提供 REST API，同时托管前端静态文件
- 下载任务使用内存字典管理（单机单用户场景足够）
- 通过任务 ID 追踪每个下载/转换任务的进度状态

### 8.5 前端页面

- 纯 HTML + CSS + JavaScript 实现，无需构建工具
- 使用 Fetch API 与后端通信
- 使用轮询方式获取下载进度（间隔 1 秒）
- 前端文件由 Flask 静态文件服务托管

## 9. 错误处理

| 场景 | 前端表现 | 后端处理 |
|------|---------|---------|
| 搜索无结果 | 显示空状态提示 | API 返回空列表 |
| 搜索关键词为空 | 按钮禁用态，输入框抖动提示 | 不发起请求 |
| 网络请求失败 | Toast 提示"网络错误，请检查连接" | API 返回 5xx |
| ffmpeg 未安装 | Toast 提示"服务器配置错误" | 启动时检测，日志记录 |
| 下载中断 | 进度条变红，显示"下载失败，请重试" | API 返回错误信息 |
| 音频转换失败 | 显示"转换失败"提示 | API 返回错误信息，保留原始文件 |

| 场景 | 处理方式 |
|------|---------|
| 搜索无结果 | 提示"未找到相关视频，请尝试其他关键词" |
| 网络请求失败 | 提示网络错误，建议检查网络连接 |
| ffmpeg 未安装 | 启动时检测，提示用户安装 ffmpeg |
| 下载中断 | 提示下载失败，建议重试 |
| 音频转换失败 | 提示转换失败，保留原始下载文件供排查 |
